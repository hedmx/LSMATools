# LSMATools_WIFS analysis 模块使用说明

> **版本**: V1.1 | **依赖**: Python 3.8+, numpy, pandas, nibabel, roifile, scipy, scikit-image

---

## 一、模块概述

`analysis` 是 LSMATools_WIFS 的独立数据分析子模块，用于评估椎体分割性能并生成论文级报告。支持**双金标准**（Roigstd + Roigstd1），可同时输出算法与两个审核者的对比指标。

### 计算的指标

| 类别 | 指标 | 缩写 |
|------|------|:--:|
| 重叠度 | Dice Similarity Coefficient | DICE |
| | Intersection over Union | IoU |
| 边界精度 | 95% Hausdorff Distance | HD95 |
| | Average Surface Distance | ASD |
| 面积一致性 | ICC(2,1) — 掩膜像素面积 | ICC_area |
| 高度一致性 | ICC(2,1) — 垂线截距高度 | ICC_height |
| 检测能力 | 检出率 (Recall) | — |

---

## 二、输入数据要求

每个病例目录需包含：

```
{case_dir}/
├── *_seg.nii.gz          # ✅ WIFS 多标签分割掩膜
├── Roigstd.zip           # ✅ 金标准 1 (审核者 A)
├── Roigstd1.zip          # ✅ 金标准 2 (审核者 B)
├── *_geom.csv            # 推荐 (形态学分析用)
└── metadata.json         # 推荐 (像素间距, 默认 0.9375mm)
```

**掩膜标签**: S1=1, L5=2, L4=3, L3=4, L2=5, L1=6, T12=7, T11=8, CANAL=15

**ROI 命名**: 文件名必须为 `S1.roi`, `L5.roi`, ..., `CANAL.roi`

## 三、使用方式

```bash
cd LSMATools_WIFS
python -m analysis.run_analysis --input-dir /path/to/batch_output
```

| 参数 | 默认值 | 说明 |
|------|:---:|------|
| `--input-dir` | 必需 | 病例目录 |
| `--spacing` | 0.9375 | 像素间距 mm |
| `--skip-geom` | — | 跳过形态学分析 |
| `--skip-report` | — | 跳过报告生成 |

Python API:

```python
from analysis import DiceEvaluator, GeomEvaluator
from analysis.report_generator import generate_report

de = DiceEvaluator(input_dir='/path/to/dir').run()
ge = GeomEvaluator(input_dir='/path/to/dir').run()
generate_report(input_dir='/path/to/dir')
```

## 五、指标计算规则

### 5.1 DICE / IoU (空间重叠度)

$$DICE = \frac{2|A \cap B|}{|A| + |B|}, \quad IoU = \frac{|A \cap B|}{|A \cup B|}$$

- **A**: 算法分割掩膜的二值图（`_seg.nii.gz` 中该标签的像素）
- **B**: 金标准 ROI 填充后的二值图（`Roigstd.zip` → roifile.coordinates() → skpoly 填充）
- 仅计算 seg 和 gold 均非空的椎体，二者皆空则跳过
- 报告: 均值 ± 标准差、中位数、Q1/Q3、min/max

### 5.2 HD95 / ASD (边界精度，单位 mm)

$$HD95 = P_{95}\big(d_A^B \cup d_B^A\big), \quad ASD = \frac{\text{mean}(d_A^B) + \text{mean}(d_B^A)}{2}$$

- $d_A^B$: 从掩膜 A 表面到掩膜 B 表面的最近距离 (scipy distance_transform_edt)
- 双边计算，对称。像素间距从 `metadata.json` 读取
- 仅计算 seg 和 gold 均非空的椎体

### 5.3 ICC(2,1) — 组内相关系数

**模型**: Two-Way Random Effects, Absolute Agreement, Single Rater

$$ICC(2,1) = \frac{MS_R - MS_E}{MS_R + (k-1)MS_E + \frac{k}{n}(MS_C - MS_E)}$$

- $k=2$ 评定者 (算法 + 金标准); $n$ = 椎体数
- 95% CI 用 F 分布法计算
- 评级: ICC ≥ 0.90 → 优, ≥ 0.75 → 良, ≥ 0.50 → 中, < 0.50 → 差

#### 面积 ICC

- **Seg 值**: `_seg.nii.gz` 掩膜中该标签的像素数
- **Gold 值**: ROI 填充掩膜中该标签的像素数
- 双方使用**完全相同的像素计数协议**，单位 pixels²
- 同时计算 Seg vs G1、Seg vs G2、G1 vs G2 三对

#### 垂线高度 ICC（两种维度）

**单列垂线高度**: 金标准质心col处做垂线，双方同协议，点敏感型指标。

**三列垂线均值高度**: 取掩膜腹-背方向 30%/50%/70% 三列分别做垂线取均值，平滑了单列采样的离散性。

> 单列→三列→面积，ICC 呈单调递增趋势，证明测量维度越高，标注者变异对 ICC 的影响越小。

### 5.4 检出率

$$Recall = \frac{TP}{TP + FN}$$

- TP: 金标准标注存在 AND 算法掩膜存在该标签
- FN: 金标准标注存在 BUT 算法掩膜缺失该标签
- 报告每个椎体的检出率 (如 33/33 = 100%)

## 六、ICC 报告规范 (论文引用)

面积 ICC 和垂线高度 ICC 分别独立报告，格式：

| 对比 | 面积 ICC [95% CI] | 垂线高度 ICC [95% CI] |
|------|:---:|:---:|
| Seg vs Gold1 | 0.9977 [0.9955, 0.9973] | 0.8984 (加权均值) |
| Seg vs Gold2 | 0.9951 [0.9907, 0.9943] | 0.8685 (加权均值) |
| Gold1 vs Gold2 (IR) | 0.9983 [0.9968, 0.9980] | — |

各椎体分项值见 `icc_area_summary.csv` 和 `icc_height_summary.csv`。

## 七、常见问题

**Q: 面积 ICC 和垂线高度 ICC 为何不同？**
A: 面积是积分量（对边界误差不敏感），高度是单列截距（对掩膜轮廓敏感）。面积 ICC 天然高于高度 ICC，这是医学影像分割的普遍规律。

**Q: 为什么垂线高度不报告"总体池化 ICC"？**
A: 不同椎体高度量级差异大（L5≈25mm, T11≈19mm, CANAL≈190mm），池化会因个体间变异放大而虚高 ICC。各椎体单独 ICC 反映同级别椎体间的真实一致性，论文应引述加权均值。

**Q: Gold1 vs Gold2 差异说明什么？**
A: Gold2 (Roigstd1) 标注更为密集，轮廓更精确，对垂线截距更敏感，因此高度 ICC 略低于 Gold1 (~0.03)。这是标注精度差异的正常现象，非算法问题。
